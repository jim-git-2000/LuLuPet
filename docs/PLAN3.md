# LuluPet PLAN3：剪切板与文件中转站

> 本文是 `docs/PLAN2.md` 之后的功能增强计划。目标是在现有 LuluPet 桌宠基础上加入 **C 快捷键剪切板面板** 和 **D 快捷键文件中转站面板**。两个面板都沿用 `F` 快捷键提醒面板的气泡式交互，不注册全局热键，不做独立普通窗口。

## Executive Summary

我要做的是 LuluPet 的第三阶段实用工具增强。前两个阶段已经完成桌宠主体、动画、托盘、设置、提醒面板、陪伴时长和多屏能力；PLAN3 不重写这些基础功能，而是在当前主窗口右侧气泡面板体系中继续增加两个轻量工具。

第一项是剪切板面板。主窗口激活时按 `C` 打开一个类似 Ditto 的简化剪切板历史面板，展示最近复制过的文本内容。首版只处理文本剪切板，保留最近 `100` 条，支持上下滚动浏览，后续可以扩展为点击条目写回系统剪切板、搜索、置顶、图片剪切板和跨会话持久化。

第二项是文件中转站面板。主窗口激活时按 `D` 打开一个类似文件夹的暂存面板。面板关联一个真实文件夹，默认放在 LuluPet 应用数据目录下。用户可以把文件拖入面板，程序把文件复制到暂存文件夹；面板展示该文件夹内的文件列表，并提供打开文件夹的入口。后续可以扩展自定义目录、删除、重命名、预览和清空策略。

我会继续沿用 PLAN2 的实现方式：Core 层放可测试的数据逻辑，App 层负责 WPF 面板、系统剪切板监听、拖拽和文件系统交互，Win32 层只放必要的剪切板监听 P/Invoke。每个 Phase 都要能独立构建和验收。

## 当前基线

当前仓库已经具备以下相关能力：

| 能力 | 当前状态 | PLAN3 如何复用 |
|---|---|---|
| 气泡式提醒面板 | 已有 `ReminderBubblePanel` | 新增两个面板视觉和交互参考它 |
| 主窗口快捷键 | `MainWindow.OnKeyDown` 已处理 `F` 等按键 | 增加 `C` 和 `D` 分支 |
| 右侧面板列 | `MainWindow.xaml` 中有 `ReminderColumn` | 改造为可容纳多个工具面板的区域 |
| 点击穿透兼容 | 点击穿透开启时快捷键面板不生效 | 剪切板/中转站保持同一规则 |
| 设置存储 | 已有 `AppSettings` + `JsonSettingsStore` | 增加工具面板配置节 |
| 应用数据路径 | 已有 `LuluPetDataPaths` | 增加文件中转站默认目录 |
| 单元测试工程 | 已有 `LuluPet.Core.Tests` | 增加剪切板历史和文件中转站逻辑测试 |

需要注意：当前 `W` 已用于 Walk 动画。PLAN3 最终使用 `D` 作为文件中转站快捷键，因此不再占用 `W`，Walk 的既有入口可以保持不变。

## 功能目标

### 1. C 快捷键剪切板面板

当 LuluPet 主窗口处于激活状态时，按 `C` 打开或关闭剪切板面板。

这里的“激活”定义为 WPF 主窗口当前获得焦点或可接收键盘事件。首版不注册全局热键，不拦截其他应用里的 `C`，也不替代系统剪切板快捷键。

剪切板面板必须满足：

| 项目 | 要求 |
|---|---|
| 入口 | 主窗口激活时按 `C` |
| 重复打开 | 如果面板已显示，再按 `C` 关闭 |
| 视觉 | 使用和 `F` 提醒面板一致的气泡式浮层 |
| 内容 | 展示最近复制过的文本剪切板内容 |
| 容量 | 保留最近 `100` 条 |
| 滚动 | 列表支持上下滚动 |
| 去重 | 连续复制相同文本时不重复堆叠 |
| 隐私 | 首版默认只在本次运行内保存历史，不写入磁盘 |
| 操作 | 点击条目可作为后续增强写回系统剪切板 |

首版只处理 `DataFormats.Text` / Unicode 文本。文件复制、图片、HTML、RTF 等格式先不纳入基础范围，避免过早引入复杂数据格式和隐私风险。

### 2. D 快捷键文件中转站面板

当 LuluPet 主窗口处于激活状态时，按 `D` 打开或关闭文件中转站面板。

文件中转站面板必须满足：

| 项目 | 要求 |
|---|---|
| 入口 | 主窗口激活时按 `D` |
| 重复打开 | 如果面板已显示，再按 `D` 关闭 |
| 视觉 | 使用和 `F` 提醒面板一致的气泡式浮层 |
| 存储 | 关联一个真实文件夹 |
| 默认目录 | 使用 LuluPet 应用数据目录下的 `Transit` 子目录 |
| 内容 | 展示暂存文件夹里的文件列表 |
| 拖入 | 用户把文件拖入面板时复制到暂存文件夹 |
| 打开 | 提供打开暂存文件夹的按钮 |
| 刷新 | 面板打开时刷新列表，拖入后立即刷新 |

首版默认采用“复制”而不是“移动”。这样拖入文件不会破坏原文件位置，风险更低；后续如需移动，可以加设置项。

### 3. 多面板互斥

`F`、`C`、`D` 三个面板都属于主窗口右侧工具面板。为避免窗口宽度失控和焦点混乱，首版采用互斥规则：

| 操作 | 行为 |
|---|---|
| 按 `F` | 打开提醒面板，并关闭剪切板/文件中转站 |
| 按 `C` | 打开剪切板面板，并关闭提醒/文件中转站 |
| 按 `D` | 打开文件中转站面板，并关闭提醒/剪切板 |
| 按当前面板快捷键 | 关闭当前面板 |
| 点击面板关闭按钮 | 关闭当前面板 |

后续如果需要并排面板，可以再扩展，但首版不做。

## 推荐实现架构

```text
src/LuluPet.Core/
  Clipboard/
    ClipboardHistory.cs
    ClipboardHistoryItem.cs
  FileTransit/
    FileTransitItem.cs
    FileTransitSnapshot.cs

src/LuluPet.App/
  Controls/
    ClipboardBubblePanel.xaml
    ClipboardBubblePanel.xaml.cs
    FileTransitBubblePanel.xaml
    FileTransitBubblePanel.xaml.cs
  Services/
    ClipboardMonitor.cs
    FileTransitService.cs

src/LuluPet.Win32/
  ClipboardNativeMethods.cs

src/LuluPet.Core/Config/
  AppSettings.cs
  ToolPanelSettings.cs

src/LuluPet.Core/Storage/
  LuluPetDataPaths.cs
```

### Core 层职责

Core 层只处理可测试、无 WPF 依赖的逻辑：

| 模块 | 职责 |
|---|---|
| `ClipboardHistory` | 维护文本历史、去重、容量限制、排序 |
| `ClipboardHistoryItem` | 描述一条剪切板历史 |
| `FileTransitItem` | 描述一个暂存文件条目 |
| `FileTransitSnapshot` | 描述暂存目录当前文件列表 |
| `ToolPanelSettings` | 保存工具面板相关设置 |

Core 层不能引用 `System.Windows.Clipboard`、WPF 控件或 Win32 API。

### App 层职责

App 层负责 UI 和桌面能力接线：

| 模块 | 职责 |
|---|---|
| `MainWindow` | 处理 `C`、`D`，显示/隐藏互斥面板 |
| `ClipboardBubblePanel` | 显示剪切板历史列表 |
| `FileTransitBubblePanel` | 显示中转站文件列表并接收拖拽 |
| `ClipboardMonitor` | 监听系统剪切板变化并读取文本 |
| `FileTransitService` | 创建目录、复制拖入文件、读取文件列表、打开文件夹 |

剪切板监听建议优先使用 Win32 `AddClipboardFormatListener`，通过主窗口句柄接收剪切板变化消息；如果实现复杂，可以临时用低频 `DispatcherTimer` 轮询作为过渡，但最终应避免高频轮询。

### Win32 层职责

Win32 层只封装必要 P/Invoke：

| API | 用途 |
|---|---|
| `AddClipboardFormatListener` | 注册剪切板变化通知 |
| `RemoveClipboardFormatListener` | 取消剪切板监听 |
| `WM_CLIPBOARDUPDATE` | 接收剪切板更新消息 |

不要把剪切板历史、UI 或文件操作放进 Win32 层。

## 配置结构

`settings.json` 建议增加 `toolPanels`：

```json
{
  "toolPanels": {
    "clipboardHistoryLimit": 100,
    "clipboardPersistEnabled": false,
    "fileTransitFolderPath": "",
    "fileTransitCopyOnDrop": true
  }
}
```

`AppSettings` 增加：

```csharp
public ToolPanelSettings ToolPanels { get; set; } = new();
```

`ToolPanelSettings` 建议字段：

```csharp
public sealed class ToolPanelSettings
{
    public int ClipboardHistoryLimit { get; set; } = 100;

    public bool ClipboardPersistEnabled { get; set; }

    public string FileTransitFolderPath { get; set; } = string.Empty;

    public bool FileTransitCopyOnDrop { get; set; } = true;
}
```

约束规则：

| 字段 | 规则 |
|---|---|
| `ClipboardHistoryLimit` | 限制在 `1` 到 `100`，非法值回退 `100` |
| `ClipboardPersistEnabled` | 首版保留字段，默认 `false` |
| `FileTransitFolderPath` | 为空时使用默认应用数据目录 |
| `FileTransitCopyOnDrop` | 首版默认 `true`，即拖入后复制文件 |

如果旧版 `settings.json` 没有 `toolPanels` 节点，加载时必须自动补默认值，并且不能丢失已有配置。

## 文件中转站目录

默认目录建议：

```text
{LuluPetDataRoot}/Transit
```

目录规则：

| 项目 | 规则 |
|---|---|
| 创建时机 | 程序启动或首次打开文件中转站时创建 |
| 缺失目录 | 自动重新创建 |
| 拖入文件 | 复制到暂存目录 |
| 同名文件 | 自动追加序号，避免覆盖 |
| 子目录 | 首版可先忽略目录拖入，只处理普通文件 |
| 文件列表 | 只展示暂存目录第一层文件 |

同名文件建议命名：

```text
report.pdf
report (1).pdf
report (2).pdf
```

两个新面板必须和提醒面板处在同一个视觉系统里，避免做成普通设置页或文件管理器窗口。

| 文件信息 | 显示文件名、大小、修改时间 |

| 面板 | 建议宽度 | 建议高度 |
|---|---:|---:|
| 提醒面板 | 沿用现有 `300px` 左右 |

## 交互规则

### 快捷键

| 快捷键 | 行为 |
|---|---|
| `F` | 打开/关闭提醒面板 |
| `C` | 打开/关闭剪切板面板 |
| `D` | 打开/关闭文件中转站面板 |
| `W` | 保持当前 Walk 行为 |

所有快捷键只在主窗口激活时生效。

### 点击穿透

如果用户开启了点击穿透：

1. `F`、`C`、`D` 快捷键都不打开工具面板。
2. 已打开的工具面板应在开启点击穿透时关闭，或至少不再接收交互。
3. 不为了打开工具面板临时关闭点击穿透。
4. 只有用户关闭点击穿透后，`F`、`C`、`D` 面板才恢复可用。

### 剪切板监听

剪切板监听规则：

1. 程序启动后开始监听系统剪切板变化。
2. 只读取文本内容。
3. 空文本、纯空白文本不进入历史。
4. 连续重复文本不新增条目，只更新时间或移动到顶部。
5. 超过容量后删除最旧条目。
6. 程序退出时停止监听。

### 文件拖入

文件拖入规则：

1. 只接受文件路径拖入。
2. 首版忽略文件夹拖入。
3. 拖入文件复制到暂存目录。
4. 复制失败时不崩溃，在面板里显示简短错误或状态。
5. 同名文件不覆盖，自动生成新文件名。
6. 复制完成后刷新列表。

### Phase P21｜文件拖入暂存

**目标**

让文件中转站支持拖入文件，并复制到关联文件夹。

**任务清单**

1. 在 `FileTransitBubblePanel` 上开启 `AllowDrop`。
2. 处理 `DragEnter` / `DragOver`，只接受文件路径。
3. 处理 `Drop`，把拖入文件传给 `FileTransitService`。
4. `FileTransitService` 复制文件到中转目录。
5. 同名文件自动生成不冲突名称。
6. 复制完成后刷新文件列表。
7. 复制失败时显示简短错误状态。
8. 为同名文件命名和目录缺失恢复补充单元测试。

**验收标准**

- 从资源管理器拖入普通文件后，文件出现在中转目录。
- 面板列表立即刷新。
- 拖入同名文件不会覆盖已有文件。
- 拖入多个文件能逐个复制。
- 拖入文件夹时首版可忽略或提示不支持。
- 复制失败不会导致程序崩溃。

**开发注意**

首版不做大文件进度条。复制大文件时如果 UI 卡顿明显，再单独引入后台任务和进度显示。

## 风险与注意事项

| 风险 | 处理 |
|---|---|
| 剪切板涉及隐私 | 首版默认只保存在内存中，不写磁盘 |
| 系统剪切板读取失败 | 捕获异常并跳过本次更新，不影响主程序 |
| 剪切板格式复杂 | 首版只支持文本 |
| 面板越来越多导致 MainWindow 复杂 | Phase P18 先抽象通用工具面板宿主 |
| 点击穿透影响面板 | 点击穿透开启时不响应 `F`、`C`、`D`，避免临时改动用户设置 |
| 文件复制覆盖 | 同名文件自动追加序号 |
| 大文件复制卡 UI | 首版接受基础复制，后续如有问题再做后台任务 |
| 拖入文件夹语义复杂 | 首版不处理文件夹拖入 |
| 默认目录被删除 | 打开中转站时自动重新创建 |
| `D` 与文本输入冲突 | 快捷键只在主窗口激活时生效；面板输入控件内如有文本输入要避免误触 |
